# mcp-servers/ekkos-memory

This directory contains the official, canonical MCP (Model Context Protocol) server for ekkOS, providing AI coding assistants with persistent memory, pattern tracking, and directive management. It acts as a secure, API-based bridge between local AI agents and the ekkOS cloud platform.

## Architecture
- **Runtime**: Node.js 18+ (TypeScript)
- **Entry points**: `index.ts` (Primary server implementation)
- **Key dependencies**: 
  - `@modelcontextprotocol/sdk`: Core framework for MCP communication.
- **Build system**: `tsc` (TypeScript compiler) via `npm run build`.
- **Deploy target**: Distributed as an npm package (`@ekkos/mcp-server`) for local execution via `npx` or global installation.

## Key Files
- `index.ts`: The main server logic; handles MCP protocol requests, API authentication, and tool execution.
- `package.json`: Defines the package metadata, dependencies, and build scripts.
- `tsconfig.json`: TypeScript configuration targeting ES2022/Node16.
- `README.md`: User-facing documentation for installation and configuration.
- `index-legacy.ts`: Deprecated entry point (do not use).

## API / Exports
The server exposes 42 tools via the MCP protocol, dynamically fetched from `https://mcp.ekkos.dev/api/v1/mcp/tools`. Key tool categories include:
- **Core Memory**: `ekkOS_Search`, `ekkOS_Forge`, `ekkOS_Directive`, `ekkOS_Context`, `ekkOS_Capture`, `ekkOS_Codebase`, `ekkOS_Recall`.
- **Plan Management**: `ekkOS_Plan`, `ekkOS_Plans`, `ekkOS_PlanStatus`, `ekkOS_PlanStep`, `ekkOS_Generate`, `ekkOS_SaveTemplate`, `ekkOS_Templates`, `ekkOS_FromTemplate`.
- **Secrets Management**: `ekkOS_StoreSecret`, `ekkOS_GetSecret`, `ekkOS_ListSecrets`, `ekkOS_DeleteSecret`, `ekkOS_RotateSecret`.
- **Portability**: `ekkOS_Export`, `ekkOS_Import`, `ekkOS_Snapshot`, `ekkOS_Sync`.

## Environment Variables
- `EKKOS_API_KEY`: Required. The user's authentication token from `platform.ekkos.dev`.
- `EKKOS_USER_ID`: Required. The unique identifier for the user.
- `EKKOS_API_URL`: Optional. Defaults to `https://mcp.ekkos.dev`.

## Conventions
- **Authentication**: Uses Bearer token authentication via the `Authorization` header.
- **Communication**: All tool calls are proxied to the ekkOS API (`/api/v1/mcp/call`) rather than executing logic locally.
- **Caching**: Tool definitions are cached for 5 minutes to reduce API overhead.
- **Error Handling**: Errors are caught and returned as `isError: true` responses within the MCP protocol to ensure the AI agent receives meaningful feedback.
- **Logging**: All logs are directed to `stderr` to prevent interference with the `stdout` MCP transport stream.

## Gotchas
- **API Dependency**: This server is a thin client; if `mcp.ekkos.dev` is unreachable, the server will fail to provide most functionality.
- **Configuration**: Users must restart their IDE after updating `mcp.json` or `settings.json` for changes to take effect.
- **Legacy Code**: `index-legacy.ts` exists in the directory but is excluded from the build; ignore it during development.
- **Tool Discovery**: If the API call to fetch tools fails, the server falls back to a minimal `ekkOS_Search` tool to prevent total system failure.
